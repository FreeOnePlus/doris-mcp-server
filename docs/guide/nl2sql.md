# NL2SQL使用指南

## 什么是NL2SQL？

NL2SQL（Natural Language to SQL）是一种将自然语言转换为SQL查询语句的技术。通过NL2SQL，您可以使用日常语言描述您的数据需求，系统会自动生成对应的SQL查询语句并执行。

## 使用技巧

### 提问的方式

为了获得最佳结果，建议按照以下方式提问：

1. **明确数据需求**：清晰描述您需要查询的内容，例如"查询2023年1月的订单总数"。

2. **指定表名**：如果您知道相关表名，请明确指出，例如"查询orders表中2023年1月的订单总数"。

3. **明确条件**：尽可能提供完整的筛选条件，例如"查询状态为已付款且金额大于1000元的订单"。

4. **指明分组和排序**：如需要分组或排序，请明确说明，例如"按产品类别分组统计销售额，并按销售额降序排列"。

### 示例查询

以下是一些有效的查询示例：

- "显示销售表中最近10条记录"
- "统计每个城市的客户数量"
- "计算上个月每个产品类别的销售总额并按销售额排序"
- "找出订单金额超过平均订单金额的客户"
- "统计男性和女性客户的数量及其年龄分布"

### 高级查询

NL2SQL还支持以下高级查询类型：

1. **子查询**：例如"查询购买过价格最高的产品的客户信息"。

2. **多表关联**：例如"显示每个客户的订单数量及消费总额"。

3. **窗口函数**：例如"计算每个产品在各月份的销售额及环比增长率"。

4. **条件分支**：例如"显示产品销量，如果销量大于100显示'热销'，否则显示'一般'"。

## 常见问题

### 系统无法理解我的查询怎么办？

如果系统无法正确理解您的查询，请尝试：

1. **重新表述**：使用不同的词语或句式表达同样的需求。
2. **分步查询**：将复杂查询拆分为多个简单查询。
3. **提供更多上下文**：说明您的业务场景和数据意义。

### 生成的SQL不符合预期怎么办？

如果生成的SQL不符合您的预期：

1. **检查数据理解**：系统可能对数据结构或业务逻辑的理解有偏差。
2. **提供反馈**：明确指出不符合预期的部分，系统会学习调整。
3. **添加约束**：明确指定某些条件或表达式的形式。

## 高级应用

### 数据探索

使用NL2SQL进行探索性数据分析：

- "了解销售数据的整体分布"
- "找出销售额异常的时间点"
- "分析客户购买行为模式"

### 业务指标计算

自动计算各类业务指标：

- "计算近30天的客户留存率"
- "分析各渠道的获客成本和转化率"
- "统计产品的复购率和客单价"

## 最佳实践

1. **先了解数据**：使用"显示表结构"等命令了解数据模型。
2. **从简单开始**：先进行简单查询，逐步添加复杂条件。
3. **审核结果**：始终检查返回的数据是否符合业务逻辑。
4. **保存有用查询**：将有价值的查询保存下来，方便复用。

通过熟练运用这些技巧，您将能够更高效地利用NL2SQL功能，从数据中获取洞察。
