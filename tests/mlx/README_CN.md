# MLX模型测试工具

这是一个用于测试MLX模型的工具，支持普通生成和流式生成模式，并可实时显示Token生成速度。该工具还支持不同的模型量化位宽，可以测试模型在不同精度下的性能表现。

## 功能特点

- 支持普通生成和流式生成两种模式
- 实时显示Token生成速度和进度
- 支持定制提示词和最大生成长度
- 支持模型量化位宽选择（16位、8位、4位、3位）
- 自动下载并量化模型
- 缓存已量化的模型以提高效率
- 彩色输出和进度条显示
- 详细的性能统计

## 安装依赖

首先安装必要的依赖包：

```bash
pip install -r requirements.txt
```

## 使用方法

### 基本用法

```bash
python test_mlx.py
```

默认会使用"如何喂养宠物狐狸？"作为提示词，加载Qwen/QwQ-32B模型。

### 参数说明

以下是可用的命令行参数：

- `--prompt`: 指定提示词，默认为"如何喂养宠物狐狸？"
- `--max_tokens`: 最大生成token数，默认为200
- `--model_path`: 模型路径或名称，默认为"Qwen/QwQ-32B"
- `--bit_width`: 模型量化位宽，选项为[16, 8, 4, 3]，默认为16（fp16）
- `--group_size`: 量化分组大小，默认为64（仅在bit_width < 16时使用）
- `--stream_only`: 仅使用流式生成模式，不运行普通生成模式
- `--update_interval`: 更新token速度的间隔（每N个token更新一次），默认为5
- `--no_color`: 禁用彩色输出
- `--cache_dir`: 模型缓存目录，默认为"./mlx_models"
- `--force_convert`: 强制重新转换模型，即使缓存已存在

### 示例

#### 使用不同的提示词：

```bash
python test_mlx.py --prompt "写一首关于春天的诗"
```

#### 使用4位量化模型：

```bash
python test_mlx.py --bit_width 4
```

#### 仅使用流式生成模式并自定义更新间隔：

```bash
python test_mlx.py --stream_only --update_interval 10
```

#### 加载不同的模型：

```bash
python test_mlx.py --model_path "mistralai/Mistral-7B-Instruct-v0.2"
```

#### 使用3位量化和不同的分组大小：

```bash
python test_mlx.py --bit_width 3 --group_size 32
```

#### 强制重新转换模型（忽略缓存）：

```bash
python test_mlx.py --bit_width 4 --force_convert
```

#### 指定自定义缓存目录：

```bash
python test_mlx.py --cache_dir "/path/to/custom/cache"
```

## 工作原理

### 模型量化流程
1. 首先，脚本会检查缓存目录中是否已有对应精度的量化模型
2. 如果没有找到缓存或使用了`--force_convert`参数，脚本会：
   - 下载原始模型（如果本地不存在）
   - 使用`mlx_lm.convert`工具将模型转换为指定的量化精度
   - 将量化后的模型保存到缓存目录
3. 最后，加载量化后的模型进行测试

### 普通生成模式
在普通生成模式下，脚本会一次性生成完整的响应，然后计算总体性能指标。适合于需要一次性获取完整结果的场景。

### 流式生成模式
在流式生成模式下，脚本会逐个token生成响应，并实时显示每个token的输出以及定期更新速度信息。这种模式更接近真实的对话体验，可以实时观察模型的生成过程。

### 量化位宽的影响
- **16位(FP16)**: 默认精度，提供最高的生成质量，但占用内存最大
- **8位(INT8)**: 中等精度和内存占用，对大多数场景质量影响很小
- **4位(INT4)**: 低精度，内存占用小，但可能对某些复杂任务的质量有影响
- **3位(INT3)**: 极低精度，内存占用最小，但质量可能显著下降

## 输出说明

脚本的输出包含以下几个部分：

1. **模型配置信息**：显示模型路径、量化位宽等配置
2. **模型转换过程**：显示模型下载和转换的实时进度
3. **加载时间**：模型加载所需时间
4. **普通生成结果**：完整的生成结果、用时和速度
5. **流式生成过程**：实时输出的文本和定期更新的速度信息
6. **性能总结**：生成完成后的总体性能报告

## 注意事项

- 第一次运行时，脚本需要下载原始模型并进行量化，这可能需要较长时间
- 量化位宽越低，模型占用内存越小，同时生成速度通常会更快
- 对于大型模型，建议使用4位或8位量化以平衡性能和质量
- 3位量化是最节省内存的选项，但可能会显著影响输出质量
- 彩色输出在某些终端可能不支持，可以使用`--no_color`参数禁用

## 故障排除

如果遇到以下问题，可以尝试相应的解决方法：

1. **模型转换失败**：
   - 检查网络连接
   - 确保有足够的磁盘空间
   - 尝试使用`--force_convert`重新转换
   - 对于大型模型，确保有足够的内存

2. **内存不足错误**：
   - 尝试使用更低的量化位宽（如4位或3位）
   - 选择更小的模型（如QwQ-1.8B而不是QwQ-7B）

3. **模型加载失败**：
   - 检查模型名称或路径是否正确
   - 确保网络连接正常
   - 尝试使用`--force_convert`重新转换模型

4. **生成质量差**：
   - 如果使用了低位宽量化，尝试使用更高的位宽
   - 对于高精度要求的任务，建议使用16位或8位

5. **生成速度慢**：
   - 对于大型模型，可以使用更低的位宽来提高速度
   - 减小`group_size`可能会提高速度，但可能影响质量
   - 使用更强大的硬件
