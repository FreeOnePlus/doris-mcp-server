# Ollama模型测试工具

这是一个用于测试Ollama模型的工具，支持普通生成和流式生成模式，并可实时显示Token生成速度。

## 功能特点

- 支持普通生成和流式生成两种模式
- 实时显示Token生成速度和进度
- 支持定制提示词和最大生成长度
- 支持温度和Top-P参数调整
- 彩色输出和进度条显示
- 详细的性能统计

## 安装依赖

首先安装必要的依赖包：

```bash
pip install -r requirements.txt
```

## 前提条件

1. 确保已安装Ollama：[https://ollama.com/download](https://ollama.com/download)
2. 确保Ollama服务正在运行（默认端口11434）
3. 确保已拉取需要测试的模型（例如：`ollama pull qwen:32b`）

## 使用方法

### 基本用法

```bash
python test_ollama.py
```

默认会使用"如何喂养宠物狐狸？"作为提示词，加载qwen:32b模型。

### 参数说明

以下是可用的命令行参数：

- `--prompt`: 指定提示词，默认为"如何喂养宠物狐狸？"
- `--max_tokens`: 最大生成token数，默认为200
- `--model`: 模型名称，默认为"qwen:32b"
- `--temperature`: 温度参数，默认为0.7
- `--top_p`: 核采样参数，默认为0.9
- `--stream_only`: 仅使用流式生成模式，不运行普通生成模式
- `--update_interval`: 更新token速度的间隔（每N个token更新一次），默认为5
- `--no_color`: 禁用彩色输出
- `--base_url`: Ollama API基础URL，默认为"http://localhost:11434"

### 示例

#### 使用不同的提示词：

```bash
python test_ollama.py --prompt "写一首关于春天的诗"
```

#### 使用不同的模型：

```bash
python test_ollama.py --model "llama3"
```

#### 调整温度参数：

```bash
python test_ollama.py --temperature 0.3
```

#### 仅使用流式生成模式并自定义更新间隔：

```bash
python test_ollama.py --stream_only --update_interval 10
```

#### 连接远程Ollama服务：

```bash
python test_ollama.py --base_url "http://192.168.1.100:11434"
```

## 工作原理

### 普通生成模式
在普通生成模式下，脚本会一次性生成完整的响应，然后计算总体性能指标。适合于需要一次性获取完整结果的场景。

### 流式生成模式
在流式生成模式下，脚本会逐个token生成响应，并实时显示每个token的输出以及定期更新速度信息。这种模式更接近真实的对话体验，可以实时观察模型的生成过程。

### 参数调整的影响
- **温度(temperature)**: 控制随机性，值越低生成越确定性，值越高生成越多样化
- **Top-P(top_p)**: 控制采样的token范围，值越低越集中于高概率token

## 输出说明

脚本的输出包含以下几个部分：

1. **模型配置信息**：显示模型名称、温度等配置
2. **加载时间**：模型加载所需时间
3. **普通生成结果**：完整的生成结果、用时和速度
4. **流式生成过程**：实时输出的文本和定期更新的速度信息
5. **性能总结**：生成完成后的总体性能报告

## 注意事项

- 确保Ollama服务正在运行，否则脚本将无法连接
- 使用前请确保已使用`ollama pull <model>`命令下载所需模型
- 模型名称必须与Ollama中的模型标识完全匹配
- 对于大型模型，生成可能需要较长时间，请耐心等待
- 彩色输出在某些终端可能不支持，可以使用`--no_color`参数禁用

## 故障排除

如果遇到以下问题，可以尝试相应的解决方法：

1. **连接错误**：
   - 确保Ollama服务已启动
   - 检查`--base_url`参数是否正确
   - 尝试使用`curl http://localhost:11434/api/tags`测试API连接

2. **模型不存在错误**：
   - 确保已使用`ollama pull <model>`下载所需模型
   - 检查模型名称是否正确（区分大小写）

3. **生成超时**：
   - 对于大型模型，可能需要增加超时时间
   - 尝试减小`max_tokens`参数值

4. **生成质量差**：
   - 尝试调整温度参数，降低温度可以让输出更加确定
   - 尝试不同的模型，如llama3、mistral等

5. **生成速度慢**：
   - 尝试使用更小的模型
   - 确保电脑满足模型的最低硬件需求
   - 如果使用GPU，确保GPU驱动已正确安装

## 性能测试

| 硬件 | 模型 | 普通生成速度 | 流式生成速度 | 内存占用 |
|------|------|--------------|--------------|----------|
| M1 Pro | qwen:1.8b | 约20 tokens/秒 | 约18 tokens/秒 | 约4GB |
| M1 Pro | llama3:8b | 约12 tokens/秒 | 约10 tokens/秒 | 约8GB |
| RTX 3080 | mistral:7b | 约40 tokens/秒 | 约35 tokens/秒 | 约14GB |
| RTX 3080 | qwen:4b | 约60 tokens/秒 | 约55 tokens/秒 | 约8GB |

*注：实际性能会根据具体硬件、模型和提示词长度有所差异*
